Google Sign-in Setup

~15 min No code No deploy Two browser tabs. Done.

Can you skip this? Yes — email + password login works fine without it. The "Continue with Google" buttons will just show an error. Do this when you're ready to let pilot sellers sign in with Google.
0 of 8 steps done
A
Google Cloud Console
Get a Client ID + Secret · ~8 min
Step 1
Open Google Cloud Console and pick your project
Open the link below. At the top-left, make sure you're in the MochiPOS project (or create one if it doesn't exist yet).
Open in a new tab → console.cloud.google.com ↗
Step 2
Set up the OAuth consent screen
In the left sidebar: APIs & Services → OAuth consent screen
  • User type: External → Create
  • App name: Mochi POS
  • User support email: your email
  • Scopes: skip (defaults are fine — just click Save and Continue)
  • Save and Continue through all steps
Step 3
Add pilot sellers as Test Users
Still on the OAuth consent screen page → scroll to Test users → + Add Users.

Add every pilot seller's Gmail address here. While the app is in Testing mode, only these people can sign in with Google (max 100). You can add more anytime.
⚠️ Important: The Gmail address you add here must exactly match the email you issue the invite code to. If they're different, sign-in will be blocked.
Step 4
Create an OAuth Client ID
Left sidebar: APIs & Services → Credentials → + Create Credentials → OAuth client ID
  • Application type: Web application
  • Name: MochiPOS web
Step 5
Add the Supabase redirect URI — then copy Client ID & Secret
Still on the Create OAuth client ID form → Authorized redirect URIs → + Add URI.

Paste this (replace <project-ref> with your Supabase project ref — found in Supabase → Project Settings → General):
https://<project-ref>.supabase.co/auth/v1/callback

Click Create. A popup shows your Client ID and Client Secret — copy both and keep this tab open.
⚠️ #1 mistake: This field (in Google Cloud) gets the Supabase URL ending in /auth/v1/callback — NOT your app's URL. They're different.
B
Supabase Dashboard
Enable Google provider + allow-list your app · ~5 min
Step 6
Enable Google in Supabase
Open your Supabase project → Authentication → Providers → Google → toggle Enable.

Paste the Client ID and Client Secret you copied from Google Cloud → Save.
Open in a new tab → supabase.com/dashboard ↗
Step 7
Add your app's callback URLs to Supabase
Still in Supabase → Authentication → URL Configuration → Redirect URLs → Add URL.

Add these three (one at a time):
https://mochipos.vercel.app/auth/callback http://localhost:3000/auth/callback https://mochipos-*.vercel.app/auth/callback

Replace mochipos with your actual Vercel project name if different. The wildcard (*) covers preview deploys.

⚠️ These are your app's URLs — different from what you put in Google Cloud. Supabase needs to know where to bounce the browser back to after Google approves.
Step 8
Done — verify it works
No redeploy needed. Google auth is live the moment you saved in Step 6.

Quick test:
  • Issue a new invite to a Gmail address you added in Step 3
  • Open the invite link → click Continue with Google → sign in as that same Gmail
  • Should land in /app with a workspace ✓
If something goes wrong
→
"Redirect not allowed" at Google screen — you put the wrong URL in Google Cloud (Step 5). It should be the Supabase URL, not your app URL.
→
"Redirect not allowed" after Google approves — you're missing your app URL in Supabase (Step 7).
→
"Doesn't match your invite" — the Gmail they signed in with doesn't match the email on the invite code. Re-issue the invite to their exact Gmail address.
→
Access blocked: App is in testing — the seller's Gmail wasn't added to Test Users (Step 3). Add it and try again.
