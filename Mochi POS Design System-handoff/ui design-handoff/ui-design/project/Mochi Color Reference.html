05 · Usage rules

Do & don't

Six rules that keep Mochi looking like Mochi when other people pick up the file.

✓
DO · Indigo for primary actions

The indigo gradient is the one big thing on a screen — primary CTA, day banner, the brand mark. One per surface.

✕
DON'T · Stack two primaries

Two indigo buttons next to each other compete. The second one should be lavender accent or ghost — let the primary lead.

✓
DO · Cream for receipts & print
MochiPOS
Plushie ×2฿840

Cream is the heritage tie-in to Meowseum booth POS. Reserve it for printable surfaces only — receipts, reports, email code boxes.

✕
DON'T · Cream in the SaaS chrome
Dashboard

Cream looks like a coffee shop blog when used as SaaS chrome. Keep the dashboard on lavender-tinted whites; cream only for "this is paper" surfaces.

✓
DO · Blush sparingly
♥
First sale!

Blush only on the mascot's cheeks, hearts, and celebration moments. If it shows up on regular UI, it stops feeling special.

✕
DON'T · Invent off-palette colors

Tailwind-bright greens, cyans, magentas, oranges don't belong in Mochi. If you need a new hue, derive it from oklch() between two existing tokens.
